Weekly Pay Calculator Program

The Weekly Pay Calculator was a significant project that allowed me to transition a manual spreadsheet into a fully functional C++ program. The program calculates and stores weekly earnings based on daily hours worked and miles driven, ensuring data persistence across sessions.

Key Features and Enhancements:

  • Data Structures: I implemented a std::map to store weekly earnings, allowing efficient retrieval and update operations based on week labels. Arrays were used to manage daily inputs such as hours worked and miles driven.
  • Algorithm Optimization: The program included robust input validation using std::stringstream to ensure accurate and reliable calculations. Additionally, I optimized file I/O operations to maintain data integrity across sessions.
  • Modular Design: I modularized the code by creating reusable functions like getDoubleInput, saveEarnings, and loadEarnings, enhancing the program's maintainability and scalability.

Reflections:

This project improved my understanding of data structures and algorithms in a real-world application. The use of std::map for storing data provided efficient access and manipulation of weekly records, while input validation ensured reliable user interactions. Future enhancements could include integrating a GUI for better user interaction and expanding the data storage capabilities to handle larger datasets using a database.

Grocery Tracker Application

The Grocery Tracker is another critical project that showcases my ability to work with databases, implement security measures, and optimize algorithms for data-driven decisions.

Key Features and Enhancements:

  • Low Stock Identification: I added a function to identify and list items with a stock count below a specified threshold, allowing store managers to make informed restocking decisions.
  • Database Integration: I introduced an SQL database to store user credentials and manage grocery inventory data, replacing the original text file system. This not only improved data management but also enhanced security by implementing a login screen.
  • Security Implementation: The addition of a login screen with credentials stored in an SQL database demonstrates my foundational knowledge of security practices in software development.

Reflections:

Enhancing the Grocery Tracker application involved significant learning, particularly in integrating SQL databases with a C++ program. This project also reinforced the importance of security in software design, leading to more reliable and secure applications. Future improvements could involve expanding database functionality and refining the user interface to improve usability.
